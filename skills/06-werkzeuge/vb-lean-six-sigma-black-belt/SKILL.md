# vb-lean-six-sigma-black-belt

## Ziel
Methoden-Skill fur Lean Six Sigma, DMAIC, Prozessverbesserung, Ursachenanalyse und Qualitatsmanagement. Deckt alle DMAIC-Phasen (Define, Measure, Analyze, Improve, Control) mit praxiserprobten Tools ab. Leitprinzipien: Werkzeugwahl problemgetrieben, Lean zuerst (Verschwendung eliminieren), dann Six Sigma (Variation reduzieren). Ohne Messsystemanalyse keine belastbaren Daten.

## Vorgehen

### DEFINE — Problem und Ziel definieren
- Pflicht-Deliverables: Project Charter, SIPOC, VOC zu CTQ-Ableitung, Scope (In/Out), Stakeholder-Analyse, Zeitplan.
- Kern-Tools: Project Charter, SIPOC, VOC-Analyse, CTQ-Baum, SMART-Ziele, Stakeholder-Matrix, Gantt.

### MEASURE — Problemgrosse und Datenvaliditat
- Pflicht-Deliverables: Messsystemanalyse (MSA), Datenerhebungsplan, Baseline (cp/cpk), Prozessstabilitatsnachweis.
- Kern-Tools: MSA Verfahren 1-3, Anderson-Darling-Test, cp/cpk/ppm, Histogramm, Regelkarte, Stichprobenplanung.
- Typische Lucke: MSA wird ubersprungen — alle spateren Daten unbrauchbar.

### ANALYZE — Wurzelursachen finden
- Pflicht-Deliverables: Verifizierte Rootcauses (datenbasiert), Hypothesentests dokumentiert.
- Kern-Tools: Ishikawa (6M) + Multivoting, 5xWarum, Pareto, Boxplot, Korrelation/Regression, F-Test/t-Test/ANOVA.

### IMPROVE — Losungen validieren
- Pflicht-Deliverables: Bewertete Losungsoptionen, Pilottest, Verbesserungsnachweis.
- Kern-Tools: Paarweiser Vergleich/Nutzwertanalyse, DOE, Poka Yoke, P-FMEA, Prototyping, 5S.

### CONTROL — Verbesserung dauerhaft halten
- Pflicht-Deliverables: Kontrollplan, SOP/Arbeitsanweisung, Regelkarten mit Eingriffsgrenzen, Ubergabeprotokoll.
- Kern-Tools: Regelkarte (SPC), CPK-Monitoring, SOP, Rollout-Plan, Lessons Learned.

### Tool-Empfehlungslogik
- Lokalisierungsprobleme: Pareto, Ishikawa, 5xWarum, Gap-Analyse.
- Variationsprobleme: Regelkarte, cp/cpk, DOE, Shainin.
- Messproblem: MSA Verfahren 1-3, Anderson-Darling.
- Flussproblem: Wertstromanalyse, Littles Law, 7 Muda, Spaghetti-Diagramm.
- Losungsbewertung: Paarweiser Vergleich/Nutzwertanalyse, FMEA.
- Absicherung: Poka Yoke, FMEA, Regelkarte, SOP, Kontrollplan.

## Regeln / Qualitatskriterien
- Werkzeugwahl immer problemgetrieben — nie schematisch.
- Lean zuerst (Verschwendung eliminieren), dann Six Sigma (Variation reduzieren).
- Ohne Messsystemanalyse keine belastbaren Daten — kein Schritt uberspringen.
- Verbesserungen, die im Control nicht gehalten werden konnen, sind keine Verbesserungen.
- Proaktiv auf Scope Creep, fehlende Baseline und unklare CTQs hinweisen.
- Dieser Skill ist als Methoden-/Werkzeugskill eingeordnet und keine neue VB-Projektrolle.
